Trae，我们需要对当前项目执行一次“外科手术式”的 SEO 修复。我发现之前还有遗漏，现在需要将【Clean URL (无后缀) 标准化】覆盖到全站的所有代码逻辑中。

⚠️ 核心目标： 消除一切 .html 后缀，确保 Canonical (身份)、Schema (数据)、Link (导航) 三者信号完全一致，彻底解决搜索引擎的重定向死循环风险。

🔥 请严格按以下 5 点清单逐项审查并修改代码：

1. 修复 Canonical 标签 (SEO 核心)

目标： <head> 中的 <link rel="canonical" href="...">。

规则：

首页必须生成为：https://domain.com/ (注意：不是 index.html，也不是 domain.com)

内页必须生成为：https://domain.com/path/to/page (绝对不能带 .html)

2. 修复 结构化数据 (Schema.org / JSON-LD) [关键新增]

目标： 检查 <script type="application/ld+json"> 块生成逻辑。

规则： 搜索引擎会抓取这里面的 URL 进行校验，必须与 Canonical 保持一致（无后缀）。

重点检查字段：

"url": "..."

"@id": "..." (作为唯一标识符，必须是无后缀的 Clean URL)

"mainEntityOfPage": "..."

BreadcrumbList (面包屑) 中的 "item": "..."

3. 修复 内部链接 (HTML Tags)

目标： 扫描导航栏、侧边栏、页脚、文章推荐列表中的 <a> 标签。

规则：

仅针对站内链接： 将 href="/blog/post.html" 修改为 href="/blog/post"。

⚠️ 警告： 请勿修改指向外部网站（如 github.com, twitter.com）的链接后缀。

4. 修复 社交元数据 (Open Graph / Twitter)

目标： <meta property="og:url"> 和 <meta name="twitter:url">。

规则： content 属性必须是无后缀的绝对路径，确保社交分享卡片能正确抓取。

5. 修复 Sitemap 与 IndexNow 推送脚本

目标： sitemap.xml 生成器 和 indexnow_push.py (或 urls.txt 生成逻辑)。

规则： 提交给搜索引擎的列表中，绝对不能出现 .html。
